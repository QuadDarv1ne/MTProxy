# MTProxy Web UI

Веб-интерфейс для мониторинга и управления MTProxy сервером.

## 📋 Обзор

Web UI предоставляет удобный интерфейс для:
- 📊 Мониторинга статистики сервера в реальном времени
- ⚙️ Управления конфигурацией
- 🔑 Управления секретами
- 📝 Просмотра логов
- 🚨 Отслеживания алертов

## 🚀 Быстрый старт

### Запуск с сервером

Web UI автоматически обслуживается встроенным HTTP сервером MTProxy (если включено).

```bash
# Запуск с REST API (необходимо для Web UI)
./mtproto-proxy --rest-port 8080 -S <secret>
```

### Доступ к Web UI

Откройте браузер и перейдите по адресу:
```
http://localhost:8080
```

Или для удалённого доступа:
```
http://<server-ip>:8080
```

## 🎨 Функции

### Dashboard

**Панели статистики:**
- Статус сервера (Running/Stopped/Error)
- Активные подключения
- Трафик (входящий/исходящий)
- Использование CPU и памяти

**Графики:**
- Подключения over time
- Трафик over time

**Автообновление:**
- Статистика обновляется каждые 5 секунд

### Конфигурация

**Управление настройками:**
- Порт сервера
- Максимальное количество подключений
- IPv6 (включить/выключить)
- Уровень логирования
- Файл логов
- Количество workers
- AES-NI (включить/выключить)

**Действия:**
- Сохранить конфигурацию
- Перезагрузить конфигурацию

### Секреты

**Управление секретами:**
- Просмотр списка секретов
- Добавление нового секрета
- Удаление секрета
- Копирование секрета в буфер обмена

**Отображаемая информация:**
- Хеш секрета (замаскированный)
- Описание
- Дата создания
- Количество использований

### Логи

**Фильтрация:**
- По уровню (ERROR, WARNING, INFO, DEBUG)
- По времени

**Отображение:**
- Временная метка
- Уровень
- Сообщение

## 🔧 Настройка

### API Key

Для доступа к API необходим ключ аутентификации.

**Способ 1: Через localStorage**
```javascript
localStorage.setItem('mtproxy_api_key', 'your-api-key');
```

**Способ 2: Через переменную окружения** (требуется настройка сервера)

### REST API Endpoints

Web UI использует следующие endpoints:

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/v1/server/status` | GET | Статус сервера |
| `/api/v1/statistics` | GET | Статистика |
| `/api/v1/config` | GET | Получить конфигурацию |
| `/api/v1/config` | PUT | Обновить конфигурацию |
| `/api/v1/secrets` | GET | Список секретов |
| `/api/v1/secrets` | POST | Добавить секрет |
| `/api/v1/secrets/{secret}` | DELETE | Удалить секрет |
| `/api/v1/logs` | GET | Логи |

## 🔒 Безопасность

### Рекомендации

1. **Используйте HTTPS** в production:
   ```bash
   # Настройте reverse proxy с SSL
   nginx + Let's Encrypt
   ```

2. **Ограничьте доступ** по IP:
   ```nginx
   location / {
       allow 192.168.1.0/24;
       deny all;
   }
   ```

3. **Используйте API ключи** для аутентификации

4. **Не открывайте Web UI** в публичный интернет без защиты

## 🐛 Troubleshooting

### Web UI не загружается

**Проверьте:**
1. Запущен ли MTProxy с REST API
2. Правильно ли указан порт
3. Доступен ли порт (firewall)

```bash
# Проверка REST API
curl http://localhost:8080/api/v1/server/status
```

### Ошибка аутентификации

**Решение:**
1. Проверьте API ключ
2. Убедитесь что ключ передан в заголовке `X-API-Key`

### Графики не отображаются

**Проверьте:**
1. Загружен ли `charts.js`
2. Есть ли данные в API ответе
3. Консоль браузера на наличие ошибок
