# Документация по .gitignore для MTProxy

## Обзор

Этот файл `.gitignore` предназначен для проекта MTProxy и обеспечивает правильное игнорирование:
- Скомпилированных файлов и объектов
- Системных и IDE файлов  
- Временных файлов и логов
- Локальных конфигураций
- Тестовых и отладочных файлов

## Структура файла

### 1. Скомпилированные файлы и объекты
```
*.o *.a *.so *.exe *.dll *.dylib
objs/ obj/ bin/ build/
```
Игнорируются все объектные файлы, библиотеки и исполняемые файлы различных форматов.

### 2. IDE и редакторы
```
.vscode/ .idea/ *.iml
*.swp *.swo *~
```
Файлы настроек различных IDE и временные файлы редакторов.

### 3. Системные файлы
```
.DS_Store Thumbs.db ehthumbs.db
```
Системные файлы macOS и Windows, которые создаются автоматически.

### 4. Временные и логи
```
*.log logs/ *.tmp tmp/
*.bak *.backup *.orig
```
Логи, временные файлы и резервные копии.

### 5. Конфигурационные файлы
```
.env *.local config.local
*.conf.bak *.conf.old
```
Локальные конфигурации и резервные копии конфигов.

### 6. MTProxy специфичные файлы
```
proxy.pid *.sock runtime.conf
test_* *_test
```
Runtime файлы и тестовые исполняемые файлы.

## Рекомендации по использованию

### Для разработчиков:
1. **Не коммитьте локальные конфиги** - используйте `*.local` файлы
2. **Проверяйте перед коммитом** - используйте `git status` для проверки
3. **Добавляйте только исходный код** - все скомпилированные файлы игнорируются

### Для администраторов:
1. **Создавайте локальные конфиги** в файлах `*.local`
2. **Храните логи отдельно** - они не попадут в репозиторий
3. **Используйте runtime файлы** для PID и сокетов

## Расширение файла

При необходимости добавления новых правил:
1. Добавляйте в соответствующую секцию
2. Комментируйте на русском языке
3. Группируйте по функциональному признаку
4. Используйте `# ================================================` как разделитель

## Тестирование

Для проверки работы .gitignore:
```bash
# Проверить статус файлов
git status

# Проверить игнорируемые файлы
git check-ignore -v <filename>

# Показать все игнорируемые файлы
git status --ignored
```

## Примечания

- Файл структурирован по категориям для удобства сопровождения
- Все комментарии на русском языке для лучшего понимания
- Учтены особенности MTProxy и кроссплатформенной разработки
- Совместим со всеми основными операционными системами
